用hugo搭建自己的博客(Windows)

March 31, 2020

前言

这几天研究了一下用Hugo搭建个人博客，并部署在GitHub上。静态网站生成器有很多，像Hexo、Jekyll、Metalsmith等等，那为什么用Hugo呢?最重要的原因还是hugo操作比较简单，配置环境也比较容易,下面是搭建博客的主要流程。

安装Hugo及环境配置

在GitHub上安装Hugo最新的Release版本。 将下载好的hugo.exe放在一个文件夹中,例如我存储在E:Hugo\hugo.exe。 之后设置Windows系统环境变量,在计算机属性中找到高级系统设置，之后点击环境变量，在Path中将hugo.exe的存储路径E:Hugo添加并保存,这样Hugo命令就可以使用了。

验证Hugo环境是否配置成功

在命令行中输入:

hugo version

如果显示hugo的版本则说明配置成功，下面我们就可以真正搭建自己的博客啦。

在本地搭建博客

生成站点

在E:Hugo新建文件夹Site, 在E:\Hugo\Site(其实在哪里创建都可以，为了好找到，在Hugo目录下创建)输入以下命令

hugo new sit yourblogname

yourblogname是你想要创建的网站的名称。 输入命令后会有以下的文件目录:

|-archetypes: 存放default.md
|-content: 你写的博客的存放位置
|-data: 存放自定义模板，导入的toml文件
|-layouts: 存放网站的模板文件
|-static: 存放js/css/img等静态资源
|-config.toml: 网站的配置文件
|-themes: 存放下载的主题

安装主题

站点生成以后，网站是什么都没有的，因此我们要安装网站的主题。

Hugo提供了丰富的主题在https://themes.gohugo.io/上。每个主题下方都提供了获取此主题的方式。
例如Hugo Future Imperfect Slim的下载方式：

git clone https://github.com/pacollins/hugo-future-imperfect-slim.git

在其中我们也可以看见主题的说明还有使用文档。

将下载好的主题移动到themes文件夹中。

生成网站

在终端输入以下命令(注意所有操作都在博客的根目录):

hugo server -t hugo-future-imperfect-slim --buildDrafts

运行成功会出现:

Web Server is available at http://localhost:1313/ (bind address 127.0.0.1)

之后在浏览器中输入localhost:1313就可以看见自己搭建好的博客网站了。

写一篇博客

在终端输入:

hugo new post/test.md

用Visual Studio Code等编辑器打开test.md，并写点内容:

---
title: "我的第一篇博客"
date: 2020-03-31T15:30:24+08:00
draft: true
---

Hello Hugo!

此时观察网站，发现已经新建了一篇博客。

将博客部署到GitHub上

在GitHub上新建仓库，注意仓库名一定是 username.github.io，其中username就是你的用户名。

生成静态网站

在命令行中输入:

hugo -t hugo-future-imperfect-slim --baseUrl='https://username.github.io'

此时发现文件目录中生成了public文件。

将生成的文件push到新建仓库中

首先进入public文件夹,输入以下命令:

git init
git add .
git commit -m 'my first commit'
git remote add origin https://github.com/username/username.github.io.git
git push -u origin master

具体的git配置教程和github相关操作在https://git-scm.com/book/zh/v2有详细说明。 之后就可以在浏览器上访问你自己的博客username.github.io
